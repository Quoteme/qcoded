1. Basic goal of Arisu

The goal of the Arisu Project is to build an easy to interact, but still very customizable AI. The whole projekt is open-source and uses the GPLv3 opensource license to give you the best control and customization for Arisu. Siri and other other proprietary software already show(ed) that artificial intelligence can help in very much ways. But you cannot customize Siri, nor any other commercial successful AI, but this can be changed! With Arisu, you get only the base of an AI that can perfectly fit your needs, if you customize it (just like Arch Linux).


2. About Arisu

Arisu is the bare base for an AI, which can be fully customized to fit your needs. She can things other AI's cannot do and also is open-source. Read more about these benefits at '2.1 Benefits of Arisu'.


2.1 Benefits of Arisu

Since Arisu uses the GLPv3 she can customized in any way you want her to be. She uses such called 'brain.txt' files, in which all the questions and answers are stored and configured. These 'brain.txt' files can be easily changed, meaning you can switch between multiple personality's, vocabulary's or even persons.Read more about changing the brain.txt at 'How to add a language'


2.2 Why the name Arisu

Arisu (Alice Mizuki 瑞城 ありす) is the name of a character in the anime: Serial Experiments Lain.
She is a (if not the only real one) friend of the main character Lain Iwakura (岩倉 玲音 Iwakura Rein) and also has some big impact on the plot of the show. For more information, read the wikipedia article.








3. How she works

Arisu uses a brain.txt file, which stores all the answers and the main responses to user asked questions. The file stores the information in an easy to use syntax:
- the questions, the user asked to get an answer to are split with a ','.
- the last question ends with a ':' and indicates the answer, Arisu will give the user.
- then follows the answer to the questions made above and this answer is then ended with an ';'

here an example: Does your tummy hurt?, hurts your tummy?: No, my tummy does not hurt.;


4. How to: install Arisu

Linux/Mac:
1. download the main Arisu core
2. extract it
3.run the main.py

Windows:
1. download and install python 3.4 for windows
2. download the main arisu windows core
3. run the main-winodws.py

Android:
First of, Arisu isn't a nativ android app and needs the scripting layer for android (sl4a) as well as python for android (py4a) installed on your device to run.
1. install sl4a first, here the download link: Download sl4a
2. download and install py4a: Download py4a
3. download and unzip the main arisu core for android.
4. move the extracted folder to storage/sl4a/scripts/ of your phone
5. start arisu-master on your device
--Alert-- Because of some bugs in the current sl4a and py4a Arisu has no way to acces the database. This means you can only use the @demo and the @p as well as the @sound tags, but not the acctual speaking.
--Alert-- Translating takes time, so there is currently only a German version of Arisu for android.


4.1 How to: add/install a language

Arisu does not come with a language pack by default, meaning: if you try to speak with her without a language pack, you will end up getting an error or creating a new brain.txt. If you do not want to start a new brain.txt and just use an already existing one, just download a new brain.txt.
!! Alert. If you want to keep an already existing brain.txt file, back it up!
After you successfully downloaded a package of your choice, unpack it and drop the containing folders/brain.txt file into the 'lang' folder of your arisu folder. After you installed a brain.txt with all the sub folders, you instantly create a 'fork' of the original language pack.


4.2 How to: start a new brain.txt

Starting a new brain.txt is much work, but rewarding. If you download a language pack, most of the questions and answers are already defined and are easily changed. If you start a new brain.txt file, you can start with another name than Arisu and let Arisu, or however he/she/it is called then give you custom questions/answers.
To create a new brain.txt, you must first delete/rename the existing brain.txt in the lang folder and all other folders depending on the other brain.txt in the lang folder (if there even are any). After that, just create a new text file, called 'brain' (> 'brain.txt') and open it with your editor of choice. start creating basic questions and answers with the syntax described in 'How she works'. If you think you made enough for a little conversation, start Arisu like described in 'How to install' and add tags and answers in the fly while conversation with Arisu. Note: starting a new brain.txt with 'brain version, bversion, bver: Lang: EN-EN || forked-release: 0.0.1 alpha;' will not hurt you (change EN-EN to your language-country).


4.3 How to: add new sentences

To add a new sentence, simply type the sentence in, and check if a dialog pops up, telling you, that the question is not indexed. To add the new sentence, tipe 'y' and press enter. Entering 'n' would cancel that, and 'w' would open a duckduckgo.com search URL.
After you entered 'y', you'll be promted with an input screen, showing you what to do. This input screen asks you to add a new 'question tag'. Just use the method showed in 'how she works', meaning, you add every tag just by typing it in and split multiple ones with a ','. The last question tag has no ',', instead has a ':'.
Following that, insert the answer Arisu should give you. You do not need to end this sentence with a ',', ':', or ';'. Just type a '.', if you even want, but it will not affect anything.
In the end, after you did all of the above are you ready to use your new fully custom sentence.


4.4 How to: add Special tags

Special tags are a good way to let Arisu respond to you. Special tag can be placed in the normal 'brain.txt' and then executed by letting Arisu say them. A good example for special tags is @Date. @Date responses with the date of the current time. The '@' indicates that it is a special tag, all special tags are defined with a '@' at the beginning. You can use a special tag, just by writing it to a response from Arisu in the brain.txt, just like this:
brain.txt: what is the date: The current date is @Date.
Result: User: |-->what is the date?
Arisu: >The current date is 09.09.1991
Here is a short list of all the special tags:

Tag Meaning
@U User Name (found in person_info.txt)
@Date Date
@Brain Shows the brain.txt
@Perso Shows the person_info.txt
@br breaks the line/starts a new line
@Clear clears the screen.
@mkdir stops arisu from creating a new subfolder.
useful if you want to stay in the same brain.txt after an input.
@AI is only used in the question. like: how are you @AI?: I'm fine.; User input with the AI name defined in 'personal_info.txt' will be changed to @AI.
@path Shows the current directory in the /lang folder.
@up Changes one folder upwards. For example:
> Hello
|--> You look ugly.
> that's not nice. # here is a @mkdir
|-->forget it. # we are now in a new brain.txt
> ok. # here is a @up, and we changed into the previous brain.txt
|--> You look beautiful. # and we can say something else.
> thank you. # and nothing happened

5.How to use Arisu

To use Arisu make sure you first installed her and installed a language pack or starded a new brain.
-Gnu/Linux:
First of, start a terminal and cd into the folder which contains the basic Arisu core. After that, start the main.py with the command 'python main.py'.
-Windows:
Start the main_windows.py with a double click.
-Windows and Gnu/Linux
The terminal should get cleaned and a few lines of text should pop up. These show your current Arisu core information. Then the welcoming message should be printed out. After that, you can start chatting. If you installed a language pack, you should get a fitting response, if you do not get one, go to the 'install a language pack' section and make sure you did everything right.
After you spoke a bit to your Arisu core, you might get presented by a text saying a sentence would be not indexed. If so, you get presented with multiple options, like adding the sentence into the brain. For that press 'y' and enter. You can now enter an answer Arisu should give you the next time you ask her the same question. Make sure you use the special tags to let the answer be more dynamic. Make sure to always use @mkdir if you want to let the user answer to a comment of Arisu.
